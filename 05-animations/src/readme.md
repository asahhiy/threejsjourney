# このプログラムで実装している内容

1. tick関数
   => 毎フレームごとに実行される関数。window.requestAnimationFrame()
   をプログラムの最後に次のフレーム呼び出しを行っている。(数列的な考え方)
   よって、１度呼び出してしまえばディスプレイのリフレッシュレート基準で無間ループが始まる。
   これがアニメーションの基本である。

2. 環境によってアニメーションの速度が変化する問題の解決法
   => この実装では、fpsが環境依存になり、アニメーションの速度が変化してしまう。
   ここでは、環境間における差を解消する方法を記す。

### Date.now()の活用

フレーム依存にするのではなく、時間依存にすることで実現。
もしfpsが低い場合 => deltaTimeはデカくなり、一度の回転量がUPする
もしfpsが高い場合 => deltaTimeは小さくなり、ちょっとずつしか動かない。
よって同じ挙動を実現することが可能となっている。
